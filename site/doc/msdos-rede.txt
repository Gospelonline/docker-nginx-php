Varredura Ping para armazendar endereço MAC  (arp -a)

"  for /l %r in (2,1,254) do ping 192.168.25.%r " 

%r - "variavel de ambiente ([iniciando em] 1, [de x em x] 1, [até] 254) - contagem de máquina a máquina até o limite de 1000 máquinas". Altere de acordo com o número de máquinas que vocę possui no seu parque.
do - "fazer"
ping - "teste de conexăo"
192.168.0.%r - "ip da rede com a variável de contagem para os hosts"
Digite “arp –a” e será listado a tabela de endereços mac com o ip correspondente.

NsLookUp: 
Envia solicitaçőes DNS para um servidor DNS por escolha.
nslookup [domínio] [servidor dns]
O comando NsLookUp permite enviar solicitaçőes DNS para um servidor. Por padrăo, se vocę năo colocar o servidor DNS, o comando utilizará aquele que é utilizado pela sua interface de rede mas vocę pode forçar a utilizaçăo de um outro servidor.
Por exemplo, para pedir ao servidor DNS 10.0.0.3 o endereço IP correspondente ao endereço www.ccm.net é: nslookup www.ccm.net 10.0.0.3

NBTSTAT: 
Atualizaçăo do cache do arquivo Lmhosts. Exibe as estatísticas do protocolo e as conexőes TCP/IP atuais utilizando NBT (NetBIOS sobre TCP/IP).
NBTSTAT [-a Nome Distante] [-A endereço IP] [-c] [-n] [-r] [-R] [-s] [S] [intervalo]

-a (estado da placa) lista da tabela de nomes da máquina distante (nome conhecido)
-A (estado da placa) lista da tabela de nomes da máquina distante (endereço IP).
-c (cache) lista o cache de nomes distantes e tidos como os endereços IP.
-n (nomes) lista os nomes NetBIOS locais.
-r (resolvidos) lista os nomes resolvidos pela difusăo e via Wins.
-R (Carga) esvaziamento e carregamento da tabela do cache de nomes distantes.
-S (Sessőes) lista a tabela de sessőes com os endereços destino IP.
-s (sessőes) lista a tabela de sessőes convertendo os endereços de destino IP em nomes de hóspedes via o arquivo hóspedes.
Exemplo: nbtstat -A @IP

NETSTAT: 
Exibe o estado da pilha TCP/IP na sua máquina local.
NETSTAT [-a] [-e] [-n] [-s] [-p proto] [-r] [intervalo]
-a exibe todas as conexăo e portas de escutas (As conexőes do lado do servidor săo normalmente inibidas).
-e exibe as estatísticas Ethernet e pode ser combinada com a opçăo -s.
-n exibe os endereços e os números de portas sob a forma numérica.
-p proto exibe as conexőes para o protocolo especificado pelo protocolo. O protocolo pode ser utilizado com a opçăo -s para exibir as estatísticas por protocolo e pode ser TCP, UDP, ou IP.
-r exibe o conteúdo da tabela do roteador.
-s exibe as estatísticas por protocolo. Por padrăo, as estatísticas em TCP, UDP e IP săo visualizadas; a opçăo -p pode ser utilizada para especificar em subconjunto do padrăo.
intervalo: exibe novamente as estatísticas selecionadas, com uma pausa de intervalo de segundos entre cada exibiçăo. Pressione Ctrl + C para parar a exibiçăo das estatísticas.
-abnov: exibe os processos que utilizam a conexăo internet (Endereço IP, porta, endereço distante e o PID do processo que utiliza a conexăo, bem como seu nome).

ARP: 
Resoluçăo dos endereços IP em endereços MAC. Exibe e modifica as tabelas de traduçőes dos endereços IP em endereços físicos utilizados pelo protocolo de resoluçăo de endereços ARP.
ARP -s adr_inet adr_eth [adr_if] 
ARP -d adr_inet [adr_if] 
ARP -a [adr_inet] [-N adr_if] 
-a exibe as entradas ARP interrogando o protocolo de dados ativos. Se adr_inet é especificado, somente os endereços de IP e físico do computador serăo exibidos. Quando mais de uma interface de rede utilizar ARP, as entradas de cada tabela ARP săo exibidas.
-g Idęntica a -a.
adr_inet precisa um endereço internet.
-N adr_if mostra as entradas ARP para o acesso ŕ interface de rede especificada por adr_if.
-d apaga o hóspede específico por adr_inet.
-s acrescenta um hóspede e associa o endereço Internet adr_inet com o endereço físico adr-eth. O endereço físico é dado sob forma de 6 octetos hexadecimais separados por hifens. A entrada é permanente.
adr_eth especifica um endereço físico.
adr_if definida, ela especifica o endereço Internet da interface cuja tabela de traduçăo dos endereços deveria estar modificada. Năo definida, a primeira interface aplicável será utilizada. 
------------------------------------------------------------------------------------
ROUTE: 
Exibe ou modifica a tabela do roteador.
ROUTE [-f] [comando [destinaçăo] [MASK máscara da rede] [gateway]
-f apaga as tabelas do roteador de todas as entradas de gateways. Utilizado conjuntamente com um dos comandos, as tabelas săo apagadas antes da execuçăo do comando.
-p mostra a entrada na tabela depois do reboot da máquina.
Comando: especifica um dos quatro comandos: DELETE (Suprime) que apaga um itinerário, PRINT (Imprime) que exibe um itinerário, ADD (Adiciona) que acrescenta um itinerário, CHANGE (Altera) que modifica um itinerário existente.
Destino: especifica um hóspede.
MASK: se a palavra chave MASK estiver presente, o parâmetro seguinte é interpretado como o parâmetro máscara da rede.
Máscara: fornecida, ela especifica o valor da máscara da sub-rede para associar esta entrada de itinerário. Năo especificada, ela assume o valor padrăo 255.255.255.255.
Gateway: especifica a Gateway.
METRIC: especifica o custo métrico para o destino.

Configurando ip fixo na placa de Rede manualmente pelo netsh:

//mostrar a interfaces:
netsh interface ip show interfaces

//Windows 10 mudar ip: 
netsh interface ipv4 set address name="Ethernet" static 192.168.7.11

//Definir um ip estático para a interface com o id 11
netsh interface ip set address name=Ethernet static 192.168.1.2 255.255.255.0 192.168.1.1

//Definir um dns primario
netsh interface ip set dns Ethernet static 192.168.200.10 

//Definir um dns 
netsh interface ip add dns Ethernet 192.168.200.10 index=1 no

//Deletar um dns
netsh interface ip delete dns Ethernet 192.168.100.10 

//Definir atribuiçao de ip automática
netsh interface ip set address name=Ethernet dhcp

//Definir busca de endereços dns automática
netsh interface ip set dns Ethernet dhcp

//Windows Vista

//apaga todos dns e inclui 1
netsh interface ip set dnsservers name="10" static 192.168.200.10
netsh interface ip set dns name="10" static 192.168.200.10 
 
//inclui dns conforme o indice
netsh interface ip add dnsserver "10" 192.168.200.10 index=1

//apaga dns
netsh interface ip delete dnsserver "10" 192.168.100.10

Para facilitar, segue o script em batch (network-address.bat):

set /P ip= "Endereço IP: "
set /P mask= "Máscara: "
set /P gw= "Gateway: "
set /P dns1= "DNS 1: "
set /P dns2=" DNS 2: "
netsh int ip set address name="Conexăo Local" source=static %ip% %mask% %gw% 1
netsh int ip set dns "Conexăo Local" static %dns1%
netsh int ip set dns "Conexăo Local" static %dns2% index=2
echo "IP: " %ip%
echo "Mask: " %mask%
echo "Gw: " %gw%
echo "DNS1: " %dns1%
echo "DNS2: " %dns2%
